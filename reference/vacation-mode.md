# Режим отпуска (логирование без ноутбука)

Когда пользователь уезжает **без ноутбука** (а оставлять его дома включённым/
разблокированным не хочет — чувствительные данные), домашний «движок» (Фаза B)
недоступен. Решение на этот случай:

## Живой счёт — приложение Claude на телефоне
- Пользователь в начале дня вставляет «карточку отпуска» в новый чат приложения
  Claude (работает по подписке, без API-ключа и без Мака) и шлёт туда фото еды.
- Claude оценивает калории/БЖУ и ведёт нарастающий остаток за день прямо в чате.
- Карточка лежит в облаке: `Meal-AI-Inbox/Отпуск-режим.md` (и копия ниже).

## Официальный учёт — по возвращении
- Те же фото пользователь сохраняет в `Meal-AI-Inbox/photos/` (Яндекс.Диск).
- Дома: «разбери инбокс» → `process-inbox` заносит весь отпуск в `data/diary.jsonl`,
  затем `nutrition-review` даёт разбор поездки.

Данные не теряются: даже без живого счёта всё накопится в инбоксе и обработается дома.

## Текст карточки (актуализировать при смене норм в profile.md)

```
Ты — мой трекер питания на сегодня. Мои дневные нормы: {ККАЛ} ккал,
белок {БЕЛОК} г, жиры {ЖИРЫ} г, углеводы {УГЛЕВОДЫ} г. Цель — {ЦЕЛЬ}.

Я буду присылать фото или описания еды. На каждое:
1. Оцени калории и БЖУ, укажи уровень уверенности (оценка по фото приблизительная).
2. Веди нарастающий итог за день и показывай остаток в формате:
   «Съедено X/{ККАЛ} ккал — осталось Y. Белок ../{БЕЛОК} · Жиры ../{ЖИРЫ} · Углеводы ../{УГЛЕВОДЫ}».
3. Тон поддерживающий, без осуждения еды. Отвечай по-русски.

Первый приём пищи пришлю следующим сообщением.
```

> Подставьте свои нормы из `profile.md` (рассчитает скилл `setup-profile`) вместо
> плейсхолдеров `{ККАЛ}`, `{БЕЛОК}`, `{ЖИРЫ}`, `{УГЛЕВОДЫ}`, `{ЦЕЛЬ}`.
