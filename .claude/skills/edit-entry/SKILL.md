# Правка и удаление записей

Дневник (`data/diary.jsonl`) и журнал веса (`data/weight.jsonl`) — append-only,
поэтому правь их аккуратно: найди нужную строку, покажи её, подтверди, применяй.

## Шаги

1. **Определи файл и запись.** По описанию пользователя найди строку в
   `data/diary.jsonl` или `data/weight.jsonl`:
   - по дате и приёму пищи («вчерашний ужин»),
   - по описанию блюда («обед с гречкой»),
   - «последняя запись» = последняя строка файла.
   Если кандидатов несколько — покажи список и попроси выбрать.

2. **Покажи текущую запись** пользователю (что именно нашлось).

3. **Спроси/применяй изменение:**
   - *Удаление:* убери строку из файла.
   - *Правка:* измени поля (граммы, продукт, вес и т. д.) и **пересчитай `total`**
     для приёма пищи. Для веса — обнови `weight_kg`/`notes`.

4. **Покажи «до → после»** и подтверди результат.

## Как редактировать JSONL безопасно

- Это построчный JSON. Меняй/удаляй только целевую строку, остальные не трогай.
- Сохраняй формат (одна валидная JSON-строка на запись, без переносов внутри).
- **Удаляя строку, не оставляй пустую строку и сохрани завершающий `\n` в конце
  файла** — иначе следующий `log-meal` допишет приём пищи на ту же строку и
  склеит записи. После удаления проверь: файл оканчивается переводом строки.
- После правки убедись, что файл по-прежнему валиден — каждая непустая строка
  парсится как JSON:
  ```bash
  python3 -c "import json;[json.loads(l) for l in open('${CLAUDE_PROJECT_DIR:-.}/data/diary.jsonl') if l.strip()];print('OK')"
  ```
  (Команду выполнять из корня проекта или убедись что `${CLAUDE_PROJECT_DIR}` задан.)

> При сомнении лучше переспросить, какую именно запись пользователь имеет в виду,
> чем удалить не ту.
