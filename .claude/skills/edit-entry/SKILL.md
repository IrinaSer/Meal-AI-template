# Правка и удаление записей

Дневник (`data/diary.jsonl`), журнал веса (`data/weight.jsonl`) и обмеры тела
(`data/measurements.jsonl`) меняются **только через хелпер**
`scripts/jsonl-edit.py` — он атомарно переписывает файл, валидирует JSON, сам
пересчитывает `total` из `items` и гарантирует завершающий `\n`. Руками
(Edit/Write) эти файлы не правь.

## Шаги

1. **Определи файл и запись.** По описанию пользователя найди строку в
   `data/diary.jsonl`, `data/weight.jsonl` или `data/measurements.jsonl`
   (Read/Grep):
   - по дате и приёму пищи («вчерашний ужин»),
   - по описанию блюда («обед с гречкой»),
   - «последняя запись» = последняя строка файла.
   Если кандидатов несколько — покажи список и попроси выбрать.

2. **Покажи текущую запись** пользователю (что именно нашлось) и подтверди
   действие.

3. **Примени хелпером:**
   - *Удаление:*
     ```bash
     python3 scripts/jsonl-edit.py delete data/diary.jsonl "<id>"
     ```
   - *Правка:* собери исправленную запись целиком (поменяй граммы/продукты/вес;
     `total` можно не считать — хелпер пересчитает из `items` сам):
     ```bash
     python3 scripts/jsonl-edit.py replace data/diary.jsonl "<id>" '<новый json одной строкой>'
     ```
   - Ключ — значение `id` (дневник) или `date` (вес/обмеры). Если под ключ
     попадает несколько записей, хелпер не тронет файл и перечислит кандидатов
     с номерами строк — повтори с ключом `line:N`.

4. **Покажи «до → после»** (хелпер печатает обе версии) и подтверди результат.

## Если python3 недоступен

Крайний случай — правь строку через Edit сам, соблюдая: одна валидная
JSON-строка на запись, пересчитай `total`, не оставляй пустых строк, сохрани
завершающий `\n` в конце файла.

> При сомнении лучше переспросить, какую именно запись пользователь имеет в виду,
> чем удалить не ту.
