# Разбор облачного инбокса

Пользователь в течение дня кидает фото еды (и заметки) с Android в облачную папку
(Яндекс.Диск), которая синхронизируется на Мак. Этот скилл разбирает накопленное.

Путь инбокса — параметр `INBOX_PATH` в `CLAUDE.md`. Внутри: `photos/` (фото еды) и
`notes.md` (текстовые записи, по одной на строку).

## Шаги

1. **Проверь инбокс.** Перечисли файлы в `INBOX_PATH/photos/` и непустые строки в
   `INBOX_PATH/notes.md`.
   - Если пусто — не падай: сообщи «новых записей в инбоксе нет» и при автозапуске
     просто обнови `ответ.md` текущим остатком (см. ниже). Заверши.

2. **Обработай каждое фото** (по возрастанию даты файла) логикой скилла `log-meal`:
   - Прочитай изображение (Read).
   - Распознай блюдо/этикетку, оцени порции по `reference/portion-guide.md`,
     **сверься с `reference/foods.csv`**.
   - Дату/время бери из метаданных или имени файла; если неясно — из времени файла.
   - Допиши строку в `data/diary.jsonl` (формат — см. `CLAUDE.md`), `source` =
     `photo`/`label`.
   - **Перенеси обработанное фото** из `INBOX_PATH/photos/` в `photos/YYYY-MM/` с
     именем `YYYY-MM-DD-meal[-N].jpg`; путь укажи в поле `photo`.

3. **Обработай заметки** из `notes.md` (например «13:40 съела банан 100 г»):
   - Каждую строку → запись логикой `log-meal`, `source:"manual"`.
   - После успешной обработки **очисти разобранные строки** из `notes.md` (или
     перенеси в `notes.archive.md`), чтобы не обработать повторно.

4. **Обнови остаток бюджета** в `INBOX_PATH/ответ.md` (логикой `nutrition-review`
   за сегодня): итог дня + остаток калорий и БЖУ. Это файл, который пользователь
   открывает на телефоне. Формат — короткий и читаемый, напр.:
   ```
   # Сегодня, 24 июня (обновлено 14:05)
   Съедено: 980 / <норма> ккал — осталось <остаток>
   Белок 62/<норма> · Жиры 41/<норма> · Углеводы 70/<норма>
   Последнее: обед (название блюда) 655 ккал
   ```

5. **Отчитайся** пользователю (если это интерактивный запуск): что обработано,
   сколько записей, остаток дня.

## Важно

- **Идемпотентность:** обрабатывай только то, что лежит в инбоксе; после переноса
  фото и очистки заметок повторный запуск ничего не задвоит.
- Сохраняй завершающий `\n` в `data/diary.jsonl` (см. конвенции в `CLAUDE.md`).
- При низкой уверенности по фото ставь `confidence:"low"` и помечай в `notes`, что
  стоит уточнить, — пользователь поправит вечером через `edit-entry`.
- Дисклеймер о приблизительности оценок остаётся в силе.
