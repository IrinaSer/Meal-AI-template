# Запись приёма пищи

Распознай еду, оцени калории и БЖУ, добавь строку в `data/diary.jsonl` и покажи
остаток дневного бюджета.

## Шаг 1. Получи изображение/описание

- Если есть фото — прочитай его через Read (модель видит картинку).
- Если только текст («съела тарелку гречки с курицей») — работай по описанию,
  `source:"manual"`.
- Если фото нет, но ожидается — попроси прислать или описать.

## Шаг 2. Определи тип и распознай

**Если это блюдо/тарелка:**
1. Перечисли продукты на фото.
2. Оцени порции в граммах по `reference/portion-guide.md`.
3. **Сверься с `reference/foods.csv`** — если блюдо знакомое, бери выверенные
   значения и `typical_grams`; пометь `source:"foods.csv"`.
4. Учти скрытые калории (масло, заправка, соус).
5. `source:"photo"` (или `"manual"`), `confidence` = `medium`/`low`.

**Если это этикетка/упаковка:**
1. Прочитай с фото пищевую ценность (на 100 г и/или на порцию) и размер порции.
2. Посчитай на фактически съеденное количество.
3. `source:"label"`, `confidence:"high"`.

## Шаг 3. Посчитай и сохрани

- Посчитай ккал и БЖУ по каждой позиции, затем `total` (сумма).
- Определи `meal` по времени суток или спроси (breakfast/lunch/dinner/snack).
- Если фото лежит по известному пути (например, из инбокса) — перемести его в
  `photos/YYYY-MM/YYYY-MM-DD-meal[-N].jpg` и укажи путь в `photo`.
- Если фото прикреплено к сообщению без файлового пути — поставь `"photo": null`
  и пометь `source:"photo"`. Фото уже проанализировано inline, сохранять не куда.
- **Допиши запись хелпером** (формат — см. `CLAUDE.md`; `total` можно не считать —
  хелпер пересчитает из `items` сам и гарантирует валидный JSONL):
  ```bash
  python3 scripts/jsonl-edit.py append data/diary.jsonl '<json одной строкой>'
  ```
  Файл руками (Edit/Write) не правь.

## Шаг 4. Отчитайся

Коротко по-русски:
- Что распознано и оценка: `≈ X ккал (Б/Ж/У: .. / .. / ..)`, уровень уверенности.
- Допущения, если есть («посчитала с учётом ложки масла»).
- **Остаток дневного бюджета:** прочитай цель из `profile.md` и сумму всех записей
  за сегодня из `data/diary.jsonl` → «осталось N ккал из M».
  - Если `profile.md` нет — запиши приём пищи, но мягко предложи запустить
    `setup-profile`, чтобы показывать остаток бюджета.

## Защита от дублирования

Если последняя запись в `data/diary.jsonl` добавлена менее 2 минут назад и её
`items` совпадают с тем, что сейчас распознаётся — переспроси: «Похоже, это уже
записано. Добавить ещё раз?»

## Коррекция

Если пользователь уточняет («порция была меньше, грамм 100», «без масла») —
пересчитай `items` и **замени только что добавленную запись**:
```bash
python3 scripts/jsonl-edit.py replace data/diary.jsonl "<id>" '<исправленный json>'
```
(`total` хелпер пересчитает сам.)

Если блюдо явно повторяющееся и пользователь подтвердил цифры — предложи занести
или обновить его в `reference/foods.csv` (значения на 100 г + `typical_grams`),
чтобы в следующий раз считать быстрее.

> Всегда честно показывай неуверенность. Лучше дать диапазон и спросить, чем выдать
> ложную точность.
