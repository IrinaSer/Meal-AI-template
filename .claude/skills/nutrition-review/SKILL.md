# Анализ питания

Сведи дневник за период, сравни с нормами, покажи паттерны и дай советы под цель
пользователя.

## Шаг 1. Определи период

Из запроса: «сегодня» (по умолчанию), «вчера», «неделя», «месяц», конкретные даты.

## Шаг 2. Собери данные

- `data/diary.jsonl` — все записи в периоде.
- `data/weight.jsonl` — взвешивания в периоде (и последнее перед ним).
- `profile.md` — дневные нормы и цель.
  - Если `profile.md` нет — мягко предложи `setup-profile`; сводку по съеденному
    дай и без норм, но без сравнения с целью.
  - Если в периоде нет записей — не падай: скажи, что данных пока нет, и предложи
    записать приём пищи.

## Шаг 3. Посчитай и сравни

- Сумма ккал и БЖУ за день; для недели/месяца — средние за день + по дням.
- Сравни со `дневными нормами` из профиля: дефицит/профицит, баланс БЖУ.
- **Динамика веса vs калораж:** если есть взвешивания — сопоставь. Вес снижается
  при дневнике с дефицитом → дефицит работает; вес стоит при заявленном дефиците →
  вероятно недооценка порций или норма завышена.

## Шаг 4. Паттерны (что искать)

- Мало белка (ниже цели) — частая проблема при похудении.
- Много добавленного сахара / жирного / фастфуда.
- Нерегулярность: пропуски приёмов, поздние/ночные перекусы.
- Однообразие, мало овощей/клетчатки.
- Дни сильно выше нормы.

## Шаг 5. Отчёт (по-русски, поддерживающий тон)

1. **Итог:** ккал и БЖУ за период vs цель (наглядно, можно «▓▓▓░░ 78%»).
2. **Тренды:** что заметно за период, динамика веса.
3. **2–4 конкретных совета** под цель похудения/здоровья — действенных и мягких
   («добавь источник белка в завтрак», «замени сладкий напиток на …»). Без
   осуждения, без жёстких диет.

## Напоминание о пересчёте нормы

Если последний вес отличается от веса в `profile.md` на `WEIGHT_RECALC_THRESHOLD_KG`
(см. `CLAUDE.md`, по умолчанию 3 кг) — напомни перезапустить `setup-profile`: по мере
похудения норма калорий снижается.

> Советы — общие принципы здорового питания, не медицинское назначение.
