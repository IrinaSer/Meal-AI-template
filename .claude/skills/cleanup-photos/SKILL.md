# Очистка старых фото

Удаляет фото из `photos/**` старше `PHOTO_RETENTION_DAYS` (см. `CLAUDE.md`, по
умолчанию 30 дней). Записи в `data/diary.jsonl` остаются нетронутыми.

## Шаги

1. Определи порог: текущая дата минус `PHOTO_RETENTION_DAYS`.
2. Найди фото старше порога (по дате из имени `YYYY-MM-DD-*` или из папки `YYYY-MM/`).
   Можно через Bash, напр.:
   ```bash
   find photos -type f \( -name '*.jpg' -o -name '*.jpeg' -o -name '*.png' -o -name '*.heic' \) -mtime +RETENTION_DAYS
   ```
   где `RETENTION_DAYS` = значение `PHOTO_RETENTION_DAYS` из `CLAUDE.md` (по умолчанию 30).
3. **Покажи итог перед удалением:** сколько файлов и какой суммарный объём
   попадают под очистку, самый старый/новый. Спроси подтверждение.
4. После подтверждения удали файлы. Удали пустые папки `photos/YYYY-MM/`.
5. Отчитайся: сколько удалено, сколько места освобождено.

> `photo`-ссылки в дневнике на удалённые файлы становятся «висящими» — это
> ожидаемо, цифры приёмов пищи сохраняются. Папка `photos/` в `.gitignore`,
> так что git это не затрагивает.
