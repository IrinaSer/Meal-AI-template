# Настройка профиля питания

Собери параметры пользователя, рассчитай нормы по формуле Mifflin-St Jeor и запиши
в `profile.md` в корне проекта.

## Шаг 1. Собери данные

Спроси (по одному или списком), если чего-то не хватает:
- Пол (м/ж)
- Возраст (лет)
- Рост (см)
- Текущий вес (кг)
- Уровень активности
- Цель: похудеть / поддерживать / набрать

Если значения уже есть в текущем `profile.md` и пользователь хочет обновить только
часть — переиспользуй остальное.

## Шаг 2. Рассчитай нормы

**BMR (Mifflin-St Jeor):**
- Мужчина: `BMR = 10·вес(кг) + 6.25·рост(см) − 5·возраст + 5`
- Женщина: `BMR = 10·вес(кг) + 6.25·рост(см) − 5·возраст − 161`

**TDEE = BMR × коэффициент активности:**
| Активность | Коэф. |
|------------|------|
| Минимальная (сидячая) | 1.2 |
| Лёгкая (1–3 тренировки/нед) | 1.375 |
| Средняя (3–5/нед) | 1.55 |
| Высокая (6–7/нед) | 1.725 |
| Очень высокая (физ. работа/спорт 2×день) | 1.9 |

**Дневная цель по калориям:**
- Похудеть: `TDEE − 15…20%` (умеренный дефицит; не опускай ниже ~1200 ккал для женщин / ~1500 для мужчин).
- Поддерживать: `≈ TDEE`.
- Набрать: `TDEE + 10…15%`.

**Цель по БЖУ:**
- Белок: `1.6–2.0 г/кг` веса (при похудении ближе к верхней границе — сохраняет мышцы).
- Жиры: `0.8–1.0 г/кг`.
- Углеводы: остаток калорий (`1 г Б = 4 ккал`, `1 г Ж = 9 ккал`, `1 г У = 4 ккал`).

Покажи пользователю расчёт и цифры, дай подтвердить.

## Шаг 3. Запиши `profile.md`

```markdown
# Профиль питания

- Пол: ж
- Возраст: 30
- Рост: 168 см
- Вес: 68 кг (на 2026-06-23)
- Активность: средняя (1.55)
- Цель: похудеть

## Дневные нормы
- Калории: 1750 ккал
- Белок: 120 г
- Жиры: 60 г
- Углеводы: 180 г

_Рассчитано по Mifflin-St Jeor. Обнови при изменении веса/цели — запусти setup-profile._
```

## Шаг 4. Запиши `data/norms.json` (машиночитаемые нормы)

Те же нормы продублируй в `data/norms.json` — одной строкой:

```json
{"kcal":1750,"protein":120,"fat":60,"carbs":180,"weight_kg":68,"updated":"2026-06-23"}
```

Из этого файла нормы читают скрипты (`statusline.sh`, `render-diary.sh`) — он
надёжнее, чем разбор Markdown. **Оба файла всегда обновляй вместе**: `profile.md` —
для человека, `norms.json` — для скриптов.

В конце скажи, что можно начинать: прислать фото еды и сказать «запиши».

> Дисклеймер: формулы дают оценку нормы, не индивидуальное медицинское предписание.
